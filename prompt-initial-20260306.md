I want to build a user-friendly web app that queries kcmo 311 tickets for potholes. The idea is to make it easier to search by location, either with an address, or latitude/longitude, and find historical 311 tickets about the pothole. there should be a small radius of tolerance that can be set, such as 50 feet, for filtering these results.

There is an OpenDataKC option. I hope it provides a connection to query for these tickets.

The idea will be for the user to be able to query for potholes that may have damaged their vehicle, and see the entire history of the pothole in terms of how many times it has been reported, when it was first/last reported, etc., and relevant 311 tickets.

Eventually, it can generate a letter to the city which documents the prior notice of the pothole and opportunity to repair the pothole (a reasonable amount of time passed), demanding compensation.
